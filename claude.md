# Piano Flex LINE構築 要件定義

## プロジェクト概要

Piano FlexのLINE公式アカウントにおいて、エルメ（L-messe）を基盤とした予約・顧客管理・セグメント配信システムを構築する。エルメの予約システムを廃止し、Googleカレンダー連携による新予約フローをLINE上で完結させる。決済はSquareリンク決済を使用。

---

## 使用ツール・サービス

| ツール | 用途 | 備考 |
|--------|------|------|
| エルメ（L-messe） | LINEシナリオ・タグ管理・リッチメニュー・配信 | **予約フロー構築完了後に廃止** |
| Googleカレンダー | レッスン日程・空き枠管理 | |
| Square リンク決済 | レッスン料金の決済 | |
| LINE公式アカウント | フロントエンド | |

> ⚠️ **移行方針**：新予約フローの構築・テスト完了を確認後、エルメを廃止する。廃止前に既存のタグ・シナリオ・リッチメニュー・挨拶メッセージのデータをエクスポート・記録しておくこと。

## 予約フロー設計（最優先実装）

### フロー概要
```
予約ボタンタップ
 ↓
① 日程選択（Googleカレンダー連携の空き枠表示）
 ↓
② 時間選択
 ↓
③ ユーザー情報入力
 ↓
④ 利用規約同意
 ↓
⑤ 料金・プラン確認メッセージ送信
 ↓
⑥ Squareリンク決済URL送付
 ↓
⑦ 予約完了メッセージ・受講済タグ付与
```

### 入力項目（③ユーザー情報入力）

| 項目 | 必須 | 備考 |
|------|------|------|
| お名前 | ✅ | |
| メールアドレス | ✅ | |
| 現時点でのお悩み | ✅ | 自由記述 |
| レッスンで特に見てほしいこと | ✅ | 自由記述 |
| その他（自由記述） | ✗ | 任意 |

### 利用規約同意（④）
- 規約テキストを表示
- 「同意する」ボタンで次へ進む
- 同意しない場合はフロー終了

## Googleカレンダー連携仕様

- レッスンの空き枠をGoogleカレンダーで管理
- 予約フロー内で空き日程をリアルタイム取得・表示
- 予約確定後、Googleカレンダーに予約情報を自動登録
- 二重予約防止のため、予約完了と同時に該当枠をブロック

## 既存設定の引き継ぎ方針

| 設定 | 方針 |
|------|------|
| タグ設計 | 既存タグを活かしつつ新タグを追加 |
| リッチメニュー | 既存デザイン・構成をベースに改修 |
| 挨拶メッセージ | 既存設定を維持 |
| 既存シナリオ | 個別に確認しながら統合 |
| エルメ全体 | 新予約フロー構築・テスト完了後に**完全廃止**。移行前にタグ・シナリオ・リッチメニュー設定を記録する |
